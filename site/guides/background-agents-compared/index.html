Background Coding Agents Compared: Assign a Ticket and Walk Away (2026)

The defining workflow of 2026: assign an AI agent a task, close your laptop, and review the pull request tomorrow morning. But which agent? Cursor ships background agents on cloud VMs with video playback. Claude Code runs headless from the terminal with unlimited parallel sessions. Devin dropped to $20/month. GitHub Copilot’s coding agent creates PRs from issues. Codex CLI sandboxes everything. This guide compares 8 platforms on the dimensions that matter: pricing, autonomy level, sandboxing, parallel execution, and how you review the output.

What Are Background Coding Agents?

A background coding agent is an AI that writes code autonomously without requiring you to watch it work. You give it a task — a GitHub issue, a Jira ticket, a plain-English description — and it works in the background: reading the codebase, writing code, running tests, and opening a pull request when done. You review the result, not the process.

This is the fundamental shift from 2025’s copilot paradigm to 2026’s agent paradigm:

  • Copilot (2023–2025): AI assists while you type. Autocomplete, inline suggestions, chat. You drive, AI navigates.
  • Background agent (2026): AI works independently. You assign, review, and merge. AI drives, you verify.

The category exploded in early 2026. Cursor shipped background agents running on isolated cloud VMs. Claude Code formalized headless mode for API-driven automation. Devin dropped from $500/month to $20/month, making autonomous agents accessible to individual developers. GitHub Copilot’s coding agent went into wide beta, turning GitHub issues into PRs with zero setup. And Codex CLI launched with the strongest built-in sandboxing of any coding agent.

The delegation gap: Anthropic’s 2026 Agentic Coding Trends Report found that developers use AI for 60% of their coding work but fully delegate (assign and walk away) only 0–20% of tasks. The gap isn’t capability — it’s trust, tooling, and workflow. Background agents with proper sandboxing, review workflows, and orchestration are how the gap closes.

Every Agent, In Depth

Cursor Background Agents $20/mo

Made by Anysphere Runs on: Cloud VMs Model: Claude, GPT-4o, custom

Cursor’s background agents are the most polished consumer-facing background agent in 2026. You describe a task in the chat panel, click “Run in Background,” and the agent spins up on an isolated cloud VM with a clone of your repo. You can close the IDE and come back later.

The standout feature is video playback — Cursor records the agent’s screen as it works, so you can watch a time-lapse of the entire coding session. This is powerful for understanding how the agent approached the problem, not just what it produced.

  • Isolation: Each task runs in its own cloud VM with full filesystem access but no access to your local machine
  • PR creation: Automatically creates a branch and PR on completion
  • Parallel: Multiple background tasks can run simultaneously, each in its own VM
  • Limitations: Cloud-only (can’t run locally), limited customization of the agent’s workflow, no API for external orchestration
Best for: Developers who live in the IDE and want the simplest possible “fire and forget” experience. The video playback alone is worth the price for teams that need to understand agent behavior.

Claude Code Headless $100/mo or API

Made by Anthropic Runs on: Your machine, CI, servers Model: Claude Opus, Sonnet, Haiku

Claude Code in headless mode (claude -p "task" --output-format stream-json) is the most flexible background agent. It runs wherever you can run a terminal — your laptop, a CI runner, a cloud server, inside Docker. No GUI, no browser, pure CLI.

The key differentiator is orchestrability. Unlike closed platforms, Claude Code headless exposes its full behavior to scripts, hooks, and orchestration tools like amux. You can spin up 10 agents in parallel, each on its own git worktree, with a shared kanban board for task coordination.

  • Isolation: Permission modes (plan mode restricts writes; auto mode requires approval for dangerous commands). Add Docker for stronger sandboxing.
  • PR creation: Not built-in, but trivial via hooks or CLAUDE.md instructions (“create a PR when done”)
  • Parallel: Unlimited. Each session is independent. Orchestrate with amux for fleet management.
  • Pricing: Max subscription ($100–$200/mo) for generous usage or API key mode at ~$3–15 per complex task
  • Limitations: No built-in video replay (session logs only), sandbox is permission-based not VM-level, requires setup for PR automation
Best for: CLI-native developers, teams running parallel agent fleets, CI/CD integration, and anyone who needs full control over the agent’s environment and workflow. The best choice for agentmaxxing. See our complete headless guide.

Claude Code Routines $100/mo (Max)

Made by Anthropic Runs on: Anthropic cloud Model: Claude Opus, Sonnet

Routines are Anthropic’s managed background agent — Claude Code running on Anthropic’s cloud infrastructure instead of your machine. You trigger a routine via the web UI, CLI (claude routine), or API, and it runs in a sandboxed cloud environment.

The advantage over self-hosted headless mode is zero infrastructure — no tmux, no server, no process management. The tradeoff is less customization and limited parallel capacity compared to running your own fleet.

  • Isolation: Cloud sandbox managed by Anthropic
  • PR creation: Automatic — connects to your GitHub repo and creates PRs
  • Parallel: Supported but limited by Max plan quotas
  • Limitations: Can’t access local tools or services, limited to Anthropic’s cloud runtime, less flexible than headless mode
Best for: Teams that want managed background agents without self-hosting. Good for scheduled tasks (nightly code reviews, dependency updates). See our Routines vs self-hosted comparison.

Devin 2.0 $20/mo

Made by Cognition Runs on: Devin cloud Model: Proprietary fine-tuned

Devin was the first dedicated autonomous coding agent and remains the most opinionated. You assign it a task via Slack, the web UI, or API, and it runs in its own sandboxed environment with a full development stack — editor, browser, terminal. Devin handles the entire loop: read spec, plan, code, test, deploy.

The 2026 pricing drop from $500/month to $20/month made Devin accessible to individual developers and dramatically changed the competitive landscape.

  • Isolation: Fully sandboxed cloud environment with its own filesystem, browser, and terminal
  • PR creation: Automatic — creates branches and PRs on completion
  • Parallel: Multiple concurrent sessions supported
  • Autonomy: Highest of any agent — Devin can use a browser, deploy to staging, and run its own test suites
  • Limitations: Closed platform (no self-hosting), proprietary model (can’t swap in your preferred LLM), limited customization of agent behavior
Best for: Full-stack feature work where you want maximum autonomy. Particularly strong for SaaS features with clear specs. The price/autonomy ratio is unmatched at $20/month.

GitHub Copilot Coding Agent $39/mo

Made by GitHub / Microsoft Runs on: GitHub cloud Model: GPT-4o, Claude (via GitHub)

Copilot’s coding agent is the most frictionless entry point to background agents. Assign a GitHub issue to @copilot and it creates a PR. That’s it. No CLI, no configuration, no separate platform. If your team already lives in GitHub, the setup cost is zero.

  • Isolation: Runs in a GitHub-hosted container (similar to GitHub Actions runners)
  • PR creation: Automatic from issue assignment — the tightest issue-to-PR pipeline of any agent
  • Parallel: Limited — GitHub manages the queue, you don’t control concurrency directly
  • Limitations: GitHub-only (doesn’t work with GitLab or Bitbucket), limited to what a GitHub Actions runner can do, less autonomy than Claude Code or Devin
Best for: Teams deep in the GitHub ecosystem who want zero-setup background agents. Perfect for triaging low-complexity issues (bug fixes, dependency updates, docs). Not ideal for complex multi-file features.

Codex CLI API credits

Made by OpenAI Runs on: Your machine Model: GPT-4o, o3, o4-mini

Codex CLI has the strongest built-in sandboxing of any coding agent. It runs with OS-level isolation: filesystem access restricted to the project directory, network disabled by default, no access to anything outside the sandbox. This makes it the safest agent to run unattended on sensitive codebases.

  • Isolation: OS-level filesystem restrictions + network disabled by default. No other agent matches this out of the box.
  • PR creation: Manual — Codex applies changes to your working directory, you decide what to commit
  • Parallel: Possible but manual — run multiple instances yourself, each in its own worktree
  • Open source: Yes — Apache 2.0 on GitHub
  • Limitations: No built-in orchestration, no PR automation, no cloud option, less autonomous than Devin or Cursor
Best for: Security-conscious teams, audited codebases, and developers who want the strongest default sandbox. Pair with amux for parallel execution and fleet management.

OpenHands Free (OSS)

Made by All Hands AI Runs on: Self-hosted Model: Any (Claude, GPT, open-weight)

OpenHands (formerly OpenDevin) is the leading open-source autonomous coding agent. Self-hosted, bring your own LLM, full control. It runs in a Docker sandbox with a browser, terminal, and file editor — similar to Devin’s architecture but fully open-source.

  • Isolation: Docker sandbox — each task runs in its own container
  • PR creation: Automatic — integrates with GitHub
  • Parallel: Supported — run multiple containers
  • LLM flexibility: Works with Claude, GPT, DeepSeek, Ollama local models, and others
  • Limitations: Requires Docker and self-hosting, steeper setup than managed platforms, performance depends on your LLM choice
Best for: Teams that need full control, want to avoid vendor lock-in, or need to run agents on air-gapped networks. The strongest open-source alternative to Devin.

Augment Code Custom pricing

Made by Augment Runs on: Augment cloud Model: Proprietary

Augment Code’s Intent product takes a different approach: you write a living specification, and Augment’s agents continuously work toward it. Instead of one-shot “do this task,” Intent maintains a spec document that evolves, with agents implementing changes as the spec is updated.

  • Differentiator: Spec-driven agent orchestration — agents work from structured specifications, not ad-hoc prompts
  • Parallel: Multiple agents can work on different parts of a spec simultaneously
  • Team focus: Designed for engineering teams, not solo developers
  • Limitations: Custom/enterprise pricing, less accessible than per-seat tools, relatively new product
Best for: Engineering teams with structured development processes who want spec-driven agent orchestration. See our spec-driven development guide for the methodology.

Running Agents in Parallel

The biggest productivity gains come from running multiple agents simultaneously — one per feature branch, one per subsystem, one per test suite. But not every platform makes this easy.

Best: Claude Code + amux

Unlimited parallel sessions with amux managing the fleet. Each agent gets its own tmux session, git worktree, and cost tracking. Shared kanban board for task coordination. Self-healing watchdog restarts crashed agents. Mobile PWA for monitoring from your phone. This is the setup for running 10+ agents overnight. See our guide.

Good: Cursor Background Agents

Multiple tasks run in parallel, each on its own cloud VM. Clean isolation, no git conflicts. Limited by your subscription’s concurrent task allowance. No external orchestration — you manage tasks through the Cursor UI.

Good: OpenHands

Run multiple Docker containers, each with its own OpenHands instance. API-driven, fully orchestrable. Requires self-hosting infrastructure and Docker resource management.

Possible: Devin 2.0

Concurrent sessions supported, but managed within Devin’s platform. Less control over orchestration than CLI-based agents. Good for 2–3 concurrent tasks; less practical for large fleets.

Possible: Codex CLI

Run multiple Codex instances manually, each in its own git worktree. No built-in orchestration — script it yourself or use amux.

Limited: Copilot Coding Agent

GitHub manages the queue. You can assign multiple issues but don’t control concurrency. Good for steady trickle of small tasks, not parallel sprints.

The Review Workflow

Background agents shift the bottleneck from writing code to reviewing it. Anthropic’s research calls this the “verification bottleneck” — the gap between generation speed and human review capacity. The review workflow you choose is as important as the agent you pick.

Morning Batch Review

Run agents overnight on decomposed tasks. Review all PRs in a 60–90 minute morning session. Works best with Claude Code + amux or Devin. See our review playbook for the complete checklist.

Continuous Triage

Agents create PRs throughout the day. Review as they come in, like reviewing a teammate’s work. Works best with Copilot Coding Agent (issues trickle in) or Cursor BG Agents (fire off tasks between meetings).

Cross-Model Review

Use one agent to write, another to review. If Cursor wrote the code, have Claude Code review the PR. Different models catch different mistakes. Automate with GitHub Actions or amux hooks.

Video Review

Cursor’s video playback and Devin’s session replay let you watch how the agent worked, not just what it produced. Use 2x playback speed for a quick sanity check before reading the diff.

How to Choose

Match the agent to your workflow, team size, and trust level.

Decision framework

  • You live in VS Code and want the simplest experience → Cursor Background Agents
  • You want a terminal-first workflow with maximum control → Claude Code Headless
  • You want managed background agents with zero infrastructure → Claude Code Routines
  • You want maximum autonomy at the lowest price → Devin 2.0 at $20/month
  • Your team lives in GitHub and wants zero setup → Copilot Coding Agent

These are not mutually exclusive. Many teams use Cursor for quick one-off tasks, Claude Code for complex multi-file features, and Copilot for triaging GitHub issues. amux can orchestrate different agents in different sessions. See our hybrid workflow guide for the Cursor + Claude Code setup.

FAQ

What is a background coding agent?

A background coding agent is an AI that writes code autonomously without requiring you to watch it in real time. You assign it a task — a GitHub issue, a Jira ticket, a plain-English description — and it works in the background: reading the codebase, writing code, running tests, and opening a pull request when done. You review the result, not the process. Major background agents in 2026 include Cursor Background Agents, Claude Code in headless mode, Devin 2.0, GitHub Copilot’s Coding Agent, Codex CLI, and OpenHands.

Which background coding agent is best in 2026?

It depends on your stack and team size. For IDE users, Cursor Background Agents offer the smoothest experience with the best UI. For CLI-native developers and parallel agent fleets, Claude Code in headless mode with amux offers the most flexibility. For fully autonomous operation on SaaS features, Devin 2.0 at $20/month is the cheapest dedicated agent. For teams in the GitHub ecosystem, Copilot Coding Agent integrates directly with issues and PRs. For open-source teams avoiding lock-in, OpenHands is the strongest self-hosted option.

How much do background coding agents cost?

Pricing varies widely. Cursor Pro is $20/month with background agents included. Claude Code with a Max subscription is $100–200/month; API-key mode costs roughly $3–15 per task. Devin 2.0 is $20/month. Copilot Coding Agent comes with Pro+ at $39/month. Codex CLI uses OpenAI API credits at ~$2–10 per task. OpenHands is free but you pay for the LLM API. See our cost calculator for detailed estimates.

Are background coding agents safe to use on production codebases?

Yes, with guardrails. Every major agent now includes sandboxing: Cursor runs agents in isolated cloud VMs, Codex CLI uses OS-level filesystem restrictions with network disabled by default, Devin runs in a sandboxed environment, and Copilot’s coding agent operates in a GitHub-hosted container. For additional safety: run agents on feature branches (never main), require PR review before merging, use pre-commit hooks, and monitor token spend. amux adds a self-healing watchdog that restarts crashed agents and prevents runaway sessions. See our sandboxing guide for deeper analysis.

Can I run multiple background agents in parallel?

Yes, and this is where the biggest productivity gains come from. Claude Code headless mode supports unlimited parallel sessions orchestrated by amux. Cursor Background Agents support multiple tasks in parallel (each in its own VM). Devin supports concurrent sessions. The key challenge is git conflict avoidance — each agent should work on its own branch or worktree. amux handles this automatically with its conflict detection and worktree isolation. See running 10+ agents.

What is the difference between a background agent and a copilot?

A copilot (like GitHub Copilot’s autocomplete or Cursor’s inline suggestions) assists you while you code — it’s reactive, waiting for your cursor position or explicit prompts. A background agent works independently — you assign a task and it executes autonomously, reading files, writing code, running commands, and creating PRs without your involvement. The shift from copilot to agent is the defining trend of 2026: developers are moving from typing with AI assistance to delegating entire tasks to AI.

How do I review code from background agents?

Treat agent PRs like PRs from a fast but inexperienced contributor. Review the diff, not the process. Check: (1) Does it solve the issue? (2) Are there tests? (3) Do tests pass? (4) Any security issues (hardcoded secrets, injection, XSS)? (5) Unnecessary dependencies? For teams running multiple agents, batch reviews in a morning session — amux’s dashboard shows all agent sessions and their PR status in one view.
