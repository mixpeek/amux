The Remote Control for Your AI Engineering Team

Keep your AI coding agents running—even when you're away from your desk. Updated July 2026.

Keep your AI coding agents running—even when you're away from your desk. The amux iOS app and PWA put your AI engineering team in your pocket: monitor live Claude Code sessions, approve pending actions, inspect terminal output, and steer agents from anywhere.

The agents run on your server. Your phone is a window into them — not a requirement for them to keep working.

What You Can Do from Your Phone

Every capability in the desktop dashboard is available on mobile. There are no reduced-functionality "mobile views."

  • Live session statuses — see which agents are actively working, waiting for input, or idle, updated in real time via Server-Sent Events
  • Peek at agent output — view any agent's last N lines of terminal output, including syntax-highlighted code blocks
  • Send instructions (Steering) — queue a message to deliver to an agent at its next turn boundary, without interrupting mid-task work
  • Kanban board — add tasks, move cards between to-do / doing / done, claim tasks atomically
  • Token spend — see how much each session has spent today, this week, and in total
  • Scheduler — view upcoming scheduled jobs, trigger them manually, enable or disable
  • Notes and CRM — read and edit notes, check contact records and follow-up reminders
  • Email — read recent messages and compose replies through the Gmail API integration

Install on iPhone (PWA)

The PWA approach works without an app store — install directly from Safari:

  1. Open https://your-server:8822 in Safari on iPhone (Safari is required for iOS PWA installation — Chrome on iOS cannot install PWAs)
  2. Tap the Share button (the box-with-arrow icon at the bottom of the screen)
  3. Scroll down and tap "Add to Home Screen"
  4. Give it a name (e.g. "amux") and tap Add
  5. The app appears on your Home Screen with the amux icon. Open it — it runs full-screen with no browser chrome.

The PWA runs in standalone mode (no address bar, no browser UI) and caches critical assets so the shell loads even when offline. Commands you send while offline queue in IndexedDB via Background Sync and replay automatically when you reconnect.

Install on Android (PWA)

  1. Open https://your-server:8822 in Chrome on Android
  2. Tap the three-dot menu in the top-right corner
  3. Tap "Install app" or "Add to Home Screen" (label varies by Chrome version)
  4. Confirm the install — the app appears in your app drawer

Android Chrome also supports the full Web Push notification API, so amux can notify you when an agent finishes a task or hits an error — without you keeping the app open.

The Native iOS App

For the best iPhone experience, download the native amux Agent Multiplexer app from the App Store:

→ amux on the App Store

The native app connects to your self-hosted amux server by URL — enter your server address on first launch. It supports both local-network connections and remote access over Tailscale or a VPN.

Remote Access: Connect from Anywhere

By default amux runs on your local network. To access your agent fleet from anywhere — from a café, on a flight, from a different country — you need secure remote access. The recommended approach is Tailscale:

  1. Install Tailscale on the machine running amux: brew install tailscale && tailscale up
  2. Install Tailscale on your phone (iOS/Android app)
  3. Your server gets a stable *.ts.net DNS name. Use that in the amux app or PWA: https://myserver.tailnet-name.ts.net:8822

amux generates a self-signed TLS certificate on first boot. Mobile browsers will show a certificate warning for self-signed certs — tap "Advanced" and "Proceed" to accept it for your private server. The iOS app handles self-signed certs transparently.

See the full remote access guide for Tailscale setup, DNS configuration, and optional reverse-proxy options.

Steering: Send Instructions Without Interrupting Your Agents

The most important mobile-native feature is the Steering queue. Steering solves a real problem: you want to give an agent new direction, but it's mid-task and interrupting it with a raw send could break its flow.

Here's how steering works:

  1. You type a message in the Steering tab of a session's peek view
  2. The message queues in SQLite — it persists even if the server restarts
  3. When the agent finishes its current response and reaches a turn boundary (ready for input), amux delivers the steering message automatically
  4. The agent gets your instruction exactly when it's ready to receive it

This is how you manage agents from your phone without babysitting them. Queue a course-correction in the morning, put your phone away, and the agent will pick it up at the right moment.

Self-Healing: Your Agents Keep Running While You're Away

The biggest anxiety about overnight or away-from-desk agent runs is: "will it just crash and sit there?" amux's self-healing watchdog eliminates this problem entirely:

  • Context overflow — when a session approaches its context limit, amux auto-compacts (using Claude Code's built-in compaction) and continues without data loss
  • Session crash — if a tmux session dies unexpectedly, the watchdog detects it within seconds, restarts the session, and replays the last message so the agent picks up where it left off
  • Stale connection — if the agent gets stuck waiting for a response that never comes, the watchdog detects the stall and intervenes

None of this requires your phone to be online. The watchdog runs continuously on the server. You can start a 12-hour agent run, put your phone in airplane mode, and come back to completed work.

The Kanban Board on Mobile

The shared kanban board is fully functional on mobile. All touch targets are at least 44×44 pixels per iOS Human Interface Guidelines. You can:

  • Add new tasks with a title, description, and assigned session
  • Drag cards between Todo / Doing / Done columns (or use the tap-to-move menu)
  • Set due dates — they appear in your calendar via the iCal feed
  • Claim tasks atomically — useful when multiple agents are competing for the next task in a queue

The board is multi-session aware: every agent session can see and update the same board, so you can coordinate a fleet of agents across multiple projects from one view on your phone.

Offline Support and Queue Replay

The amux PWA is built for intermittent connectivity. When your phone goes offline:

  • The cached dashboard shell loads instantly
  • Previously loaded session data is still visible
  • Commands you send go into an offline queue (Background Sync / IndexedDB)
  • When connectivity returns, queued commands replay automatically — no manual retry needed

This is particularly useful when managing agents across timezone transitions: queue up morning-review commands before boarding a flight, and they'll deliver when you land and reconnect.

Frequently Asked Questions

Can I really manage AI coding agents from my phone?

Yes. amux is specifically designed for this. You can see every agent's live status, peek at what each agent is doing, send new instructions, manage your task board, and check token costs — all from your phone. The dashboard is a responsive PWA that works in any mobile browser, plus there's a native iOS app on the App Store.

Does my phone need to stay on for the agents to keep running?

No. The agents run entirely on the server. Your phone is a remote control, not a host. You can close the app, turn off your phone, or lose connectivity — the agents keep running. The self-healing watchdog on the server handles crashes, context overflows, and stalls automatically.

How is amux different from just SSH-ing into my server from my phone?

SSH gives you one terminal. amux gives you a fleet dashboard: status cards for all sessions at once, a shared kanban board, cost tracking, notes, CRM, email, and a steering queue. You can also share the amux dashboard with team members — a collaborator can see agent statuses and add board tasks without needing SSH credentials. The experience is optimized for touch and mobile screens, not repurposed from a terminal emulator.

Is the iOS app free?

amux itself is free and open-source (MIT license). The iOS app is a free download on the App Store. You self-host the amux server on your own machine — you pay nothing to amux. You pay only for the AI tokens your agents consume (Claude, Codex, or Gemini CLI).

Can I use amux without a fixed server? What about running it from a laptop?

Yes. The amux server runs on any machine with Python 3 and tmux — a laptop, a desktop, a Raspberry Pi, or a cloud VM. The key for phone access is that the server stays on while your agents run. Many users run amux on a dedicated cloud VM (a $5–10/month small instance) for 24/7 access, but a laptop left open at home with Tailscale works fine for most use cases.

git clone https://github.com/mixpeek/amux && cd amux && ./install.sh
amux register myproject --dir ~/Dev/myproject --yolo
amux start myproject
amux serve  # → https://localhost:8822
