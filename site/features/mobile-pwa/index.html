Mobile PWA & iOS App
Manage AI Agents from Your Phone (2026)

amux is the only AI agent dashboard built phone-first. Install the PWA or native iOS app to monitor agents, peek terminal output, send instructions, and manage the board — all from a 375px screen.

Last updated: July 11, 2026

Quick answer: amux ships a full-featured mobile experience — a progressive web app (PWA) installable on iOS and Android, plus a native iOS app on the App Store. You get live terminal output, plan strip, steering bar, board, notes, CRM, and scheduler from your pocket. It’s the only AI agent dashboard designed phone-first.

Two ways to use amux on mobile

Native iOS App

The amux – Agent Multiplexer native app (App Store id6760410435) is built in native SwiftUI. It connects to your local amux server over your home Wi-Fi or Tailscale network and gives you full dashboard access including the peek panel, plan strip, board, and steering bar — with a native feel the browser can’t match.

Progressive Web App (PWA)

Open http://<your-laptop-ip>:8822 in Safari (iOS) or Chrome (Android) and tap “Add to Home Screen.” The PWA installs as a standalone app icon, launches full-screen, and queues actions offline via Background Sync. No App Store required.

What you can do from your phone

  • Scan the session fleet — all agents, live status dots, last active timestamp
  • Tap any session → open peek panel → read live terminal output + plan strip
  • Switch between main agent and subagent transcripts (↑/↓ arrows, tap agent rows)
  • Load earlier output — scroll back through the complete session history
  • Send instructions via the steering bar or saved message chips
  • Move tasks on the kanban board (todo → doing → done)
  • Add notes and CRM contacts on the go
  • View and manage the scheduler (recurring tasks, cron triggers)

Phone-first design details

  • Touch targets ≥44×44px — every button and tappable row meets Apple Human Interface Guidelines
  • env(safe-area-inset-*) — fixed overlays extend to physical screen edges on iOS PWA; no content hidden behind notch or home indicator
  • Responsive peek panel — measures real character width in the pane, reflows Claude Code terminal output to fit the phone screen (no horizontal scrolling)
  • Mobile-optimized bottom sticky bar — prominent iOS app download prompt on mobile widths (≤600px)
  • Offline-capable — Background Sync queues actions when you’re offline; they replay automatically when connection returns

How to connect from outside your home network

  • Tailscale (recommended for personal use) — install Tailscale on both your laptop and phone, join the same tailnet, connect to your laptop’s Tailscale IP on port 8822
  • amux tunnel (cloud feature) — run amux tunnel start 8822; your phone hits https://<id>.t.amux.io/ — stable, HTTPS, no port forwarding required. Requires amux cloud.
  • Remote server — run amux on a cloud VPS; connect to the public IP directly

Frequently asked questions

Is there a native iOS app for amux?

Yes. Search “amux Agent Multiplexer” on the App Store or open this link directly (id6760410435). The native SwiftUI app connects to your local amux server over Wi-Fi or Tailscale.

Can I install amux as a PWA on Android?

Yes. Open your amux dashboard URL in Chrome for Android and tap the three-dot menu → “Add to Home Screen.” You get a full-screen, installable PWA with offline action queuing via Background Sync.

How do I connect the amux iOS app to my laptop at home?

Run amux serve on your laptop. On the same Wi-Fi network, open the iOS app and enter http://<your-laptop-ip>:8822. For remote access when away from home, use Tailscale (recommended) or amux tunnel.

Can I manage my agents from my phone while they run overnight on my laptop?

Yes — this is a primary use case. Start agents in detached mode with amux start --detach, leave your laptop running, and monitor, steer, and review output from your phone at any time. The peek panel shows live terminal output and the plan strip.

Does the amux PWA work offline?

The action queue works offline. Instructions you send while offline are stored locally and replayed via Background Sync when your connection returns. Live terminal output and real-time agent status require an active connection to your amux server.
